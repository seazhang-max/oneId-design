## OneID 应用场景

OneID 作为用户身份的统一标识，其核心价值在于**打通身份孤岛**，为上层业务提供"一个用户、一个视角"的能力。以下是主要应用场景：

---

### 场景一：跨品牌用户识别与统一画像

**痛点**：同一自然人在集团下品牌 A 和品牌 B 分别注册了账号，品牌 A 只知道"用户 A1"，品牌 B 只知道"用户 B1"，无法识别为同一人。

**OneID 解法**：

- **集团层 OneID** 将品牌 A 的"用户 A1"和品牌 B 的"用户 B1"识别为同一自然人
- 聚合该用户在集团内所有品牌的消费行为、偏好、会员等级，形成完整的集团级用户画像
- 品牌层 OneID 保持不变，各品牌仍独立管理自己的用户

**业务价值**：

- 集团可统计跨品牌去重后的真实用户规模（而非各品牌用户数简单相加）
- 识别高价值跨品牌用户，制定集团级 VIP 策略
- 发现品牌间的用户流转路径（如品牌 A 用户是否转化为品牌 B 用户）

---

### 场景二：精准营销与个性化推荐

**痛点**：营销系统基于单品牌账号投放，同一用户在不同品牌收到重复或无关的营销信息。

**OneID 解法**：

- 通过 OneID 关联的 `AIProfile`（购买偏好、品类亲和度、价格敏感度、生命周期阶段等），实现精准分群
- 基于 `lifecycle_stage` 区分新客、活跃、沉默、流失用户，匹配不同营销策略
- 基于 `customer_segment` 区分高价值、成长型、高风险、低活跃用户，差异化运营

**业务价值**：

- 避免跨品牌重复触达，降低营销成本和用户打扰
- 根据用户跨品牌偏好做交叉推荐，提升转化率
- 流失预警（`churn_probability`）驱动主动挽留

---

### 场景三：跨渠道用户行为串联

**痛点**：同一用户可能在微信小程序、APP、PC 网站、线下门店等多渠道产生行为，各渠道数据割裂。

**OneID 解法**：

- 通过 `IdMapping` 将同一自然人的 OpenID（微信）、设备 ID（APP）、Cookie ID（Web）、手机号（线下）关联到同一个 OneID
- 串联用户在全渠道的浏览、下单、支付、售后行为

**业务价值**：

- 构建完整的用户行为旅程（User Journey），支持全链路归因分析
- 识别用户的主活跃渠道，优化渠道资源投入
- 支持全渠道一致的个性化体验（如 APP 上看到的商品，在小程序上也能看到相关推荐）

---

### 场景四：用户生命周期管理

**痛点**：无法准确判断用户从"新客 → 活跃 → 沉默 → 流失"的演变，缺乏干预依据。

**OneID 解法**：

- `AIProfile.lifecycle_stage` 基于全渠道、全品牌的行为数据计算生命周期阶段
- `AIProfile.churn_probability` 预测流失风险
- `AIProfile.purchase_intent` 预测购买意向

**业务价值**：

- 新客阶段：触发新手引导、首单优惠
- 活跃阶段：推荐高价值商品，提升客单价
- 沉默阶段：发送唤醒优惠券、推送个性化内容
- 流失阶段：高成本挽留手段（专属客服、大额优惠）
- 基于 `predicted_lifetime_value` 差异化投入挽留资源

---

### 场景五：跨集团用户识别（全局层）

**痛点**：SaaS 平台服务多个集团，同一自然人可能在不同集团的店铺消费，平台层面无法识别。

**OneID 解法**：

- **全局层 OneID** 将跨集团的同一自然人识别为同一个全局 OneID
- 通过 `CrossLayerMapping` 表维护品牌层 → 集团层 → 全局层的映射关系

**业务价值**：

- 平台可统计全局去重用户规模，评估真实市场覆盖
- 跨集团的消费能力评估，支持平台级用户分层
- 为平台级营销活动（如大促）提供去重后的用户触达

---

### 场景六：数据合规与隐私管理

**痛点**：用户发起"删除我的数据"请求时，需要找到该用户在所有品牌、所有系统中的数据。

**OneID 解法**：

- OneID 通过 `IdMapping` 记录了用户所有的 ID 标识，可快速定位所有关联数据
- 通过 `CrossLayerMapping` 可追溯用户在各层级的身份映射
- `source` 和 `attribute_sources` 字段支持数据来源溯源

**业务价值**：

- 满足 GDPR/个人信息保护法 的数据主体请求（DSR）
- 快速定位并删除/导出某用户的全部数据
- 审计溯源：追踪每个属性字段的数据来源，满足合规审计要求

---

### 场景七：实时用户识别与决策

**痛点**：新用户首次访问时，无法快速识别其身份和提供个性化服务。

**OneID 解法**：

- 实时链路通过 Flink 在秒级生成临时 OneID
- 基于设备 ID、Cookie ID 等即时可用标识，快速匹配已有 OneID 或创建新 OneID
- 次日离线 WCC 校准融合，保证最终一致性

**业务价值**：

- 老用户回访时，即使未登录也能通过设备 ID 识别身份，展示个性化内容
- 新用户首次访问即可基于实时行为开始构建画像
- 实时风控：识别异常设备/ID 聚集，防范刷单、薅羊毛

---

### 场景八：会员积分与权益打通

**痛点**：同一自然人在集团下不同品牌分别拥有会员身份和积分，体验割裂。

**OneID 解法**：

- 集团层 OneID 关联用户在各品牌的会员等级（`membership_level`）和积分（`membership_score`）
- 通过 `PhysicalInfo` 中的交易统计（累计订单数、累计消费金额）评估用户整体价值

**业务价值**：

- 支持集团级统一会员体系（如集团金卡 = 旗下任意品牌消费达标）
- 积分跨品牌兑换或累计
- 统一客服视角：客服人员通过 OneID 看到用户在所有品牌的服务记录

---

### 场景九：反欺诈与风控

**痛点**：欺诈用户在多品牌/多渠道注册大量账号薅羊毛，单品牌视角难以识别。

**OneID 解法**：

- WCC 连通分量算法天然能发现 ID 标识的聚集关系
- 同一设备 ID / Cookie ID 关联大量不同手机号 → 疑似批量注册
- `IdMapping.status` 中的 `"suspicious"` 标记支持风控标注

**业务价值**：

- 识别"羊毛党"群体（多个账号共享相同设备/地址）
- 跨品牌欺诈行为联防（某用户在品牌 A 有欺诈记录，品牌 B 可提前预警）
- 基于图结构发现异常社区（异常密集的 ID 关联子图）

---

### 场景十：经营分析与决策支持

**痛点**：各品牌独立统计用户数、GMV 等指标，集团层面数据口径不一致。

**OneID 解法**：

- 三层 OneID 提供统一的用户计数基准（品牌层去重、集团层去重、全局层去重）
- `PhysicalInfo` 中的聚合统计值支持多维分析

**业务价值**：

- 集团可准确回答"我们有多少真实用户"（而非各品牌注册用户数之和）
- 分析跨品牌用户贡献："同时购买品牌 A 和品牌 B 的用户，LTV 是多少？"
- 支持 OLAP（ClickHouse）多维分析，按层级、品牌、用户分群等维度交叉分析

---

### 应用场景总览


| 场景      | 主要使用层级    | 核心 OneID 能力 | 关键数据字段                                             |
| ------- | --------- | ----------- | -------------------------------------------------- |
| 跨品牌用户识别 | 集团层       | 跨品牌身份关联     | `CrossLayerMapping`, `IdMapping`                   |
| 精准营销    | 品牌层 + 集团层 | 用户画像 + 分群   | `AIProfile`, `PhysicalInfo`                        |
| 跨渠道行为串联 | 品牌层       | 多渠道 ID 关联   | `IdMapping`（多 source_system）                       |
| 生命周期管理  | 品牌层 + 集团层 | AI 预测 + 状态机 | `AIProfile.lifecycle_`*, `churn_probability`       |
| 跨集团识别   | 全局层       | 全局身份去重      | `CrossLayerMapping`（group_to_global）               |
| 数据合规    | 三层        | 身份溯源 + 映射追溯 | `source`, `attribute_sources`, `CrossLayerMapping` |
| 实时用户识别  | 品牌层       | 实时链路 + 离线校准 | 实时临时 OneID → 离线正式 OneID                            |
| 会员权益打通  | 集团层       | 跨品牌会员关联     | `PhysicalInfo.membership_*`                        |
| 反欺诈风控   | 集团层 + 全局层 | 图结构异常发现     | `IdMapping.status`, WCC 连通子图                       |
| 经营分析    | 三层        | 统一用户计数基准    | 三层 OneID + `PhysicalInfo` 聚合统计                     |
